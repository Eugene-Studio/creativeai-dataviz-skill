# Contributing

Thanks for considering a contribution. This skill is small and opinionated, but it benefits from real-world feedback.

## What's most useful

**Bug reports.** If a chart renders incorrectly on a specific WordPress theme, a Chart.js version conflict breaks the embed, or the workflow produces inconsistent French phrasing, open an issue with the article URL, the chosen idea, and the produced HTML.

**Style guide forks.** If you adapt this skill to another publication, please open a discussion or send the link. Forks are the best evidence the system is portable, and they help future adopters pick a starting point closer to their needs.

**Workflow improvements.** If you find that the five-concept format misses common dataviz angles, propose new concept categories with examples drawn from real articles. Editorial improvements ship faster than code changes here.

## What's out of scope

**Chart library swaps.** The skill is built around Chart.js because it's lightweight, theme-friendly, and trivial to enqueue in WordPress footers. PRs that switch to D3, ECharts, or Plotly will be declined for v1.

**English-language outputs.** The skill is French-first by design. An English variant would be a separate skill, not a flag inside this one.

**Brand-specific style tokens in this repo.** This repo's `style-guide.json` belongs to CreativeAI.fr. Other brands fork the repo and replace the file in their own copy.

## How to propose a change

1. Open an issue first to discuss the direction. This avoids work on something I won't merge.
2. For documentation fixes (typos, broken links, clarifications), a direct PR is fine.
3. For workflow changes to `SKILL.md`, please include before/after examples on a real article.

## Code of conduct

Be considerate. This is a small project run by one person. Polite disagreement is welcome; hostility gets the issue closed without further reply.
